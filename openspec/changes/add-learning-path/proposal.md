# 变更：添加结构化学习路径

## 原因
目前系统将内容视为扁平的视频列表，缺乏结构化的进阶路径。用户无法遵循精心设计的学习路径（课程 -> 单元 -> 课时），无法追踪完成状态，也无法进行互动练习。这限制了应用作为结构化学习工具的有效性。

## 变更内容
- **课程结构**: 引入 `Course`（课程）、`Unit`（单元）和 `Lesson`（课时）实体，以层级方式组织内容。
- **学习进度**: 实现进度追踪系统，记录用户在课程、单元和课时层级的进度（例如：锁定、进行中、已完成）。
- **练习模式**: 增加对互动练习会话的支持（例如：跟读、听力），并存储用户的提交结果。

## 影响
- **数据库**: 
  - 新增表: `courses`, `units`, `lessons`, `user_progress`, `practice_submissions`。
  - 修改: `videos` 表可能需要关联到 `lessons`。
- **API**:
  - 新增课程管理的 CRUD 接口。
  - 新增获取大纲/结构的接口。
  - 新增更新和获取用户进度的接口。
  - 新增提交练习结果的接口。
